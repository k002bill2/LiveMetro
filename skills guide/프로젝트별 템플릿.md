## React Native/Expo 모바일 앱 (LiveMetro 기반)

이 템플릿은 LiveMetro 프로젝트를 기반으로 한 React Native/Expo 앱의 표준 구성이다. 서울 지하철 실시간 도착정보 앱으로, Firebase 백엔드와 TypeScript strict 모드를 사용한다.

### CLAUDE.md
```markdown
# LiveMetro Project Context

## Project Overview
- **Name**: LiveMetro
- **Tech Stack**: React Native 0.72.10, Expo SDK ~49, TypeScript ^5.1.3+ (strict)
- **Backend**: Firebase Auth, Firestore
- **Navigation**: React Navigation 6.x
- **State Management**: AuthContext + Custom Hooks (Redux 미사용)
- **Testing**: Jest, React Testing Library
- **Data Flow**: Seoul Metro API -> Firebase -> AsyncStorage (Cache)

## Project Structure
\`\`\`
src/
├── components/       # 재사용 가능한 UI 컴포넌트
├── screens/          # 화면 컴포넌트
├── navigation/       # React Navigation 설정
├── services/         # API 및 Firebase 서비스
├── models/           # 데이터 모델 및 타입 정의
├── hooks/            # 커스텀 React 훅
└── utils/            # 유틸리티 함수
\`\`\`

## Path Aliases
| Alias | Path |
|-------|------|
| `@` | `src/` |
| `@components` | `src/components` |
| `@screens` | `src/screens` |
| `@services` | `src/services` |
| `@models` | `src/models` |
| `@utils` | `src/utils` |
| `@hooks` | `src/hooks` |

**반드시 경로 별칭을 사용한다. 상대 경로 import 금지.**

## Development Guidelines
- TypeScript strict 모드: `any` 사용 금지, 명시적 반환 타입 필수
- useEffect cleanup 함수에서 모든 구독/타이머 정리
- Seoul Metro API 호출 최소 30초 폴링 간격 준수
- 에러 처리: throw 대신 빈 배열/null 반환
- 커버리지 기준: statements 75%, functions 70%, branches 60%

## Navigation Structure
\`\`\`
RootNavigator -> Main (BottomTabs) -> Home | Favorites | Alerts | Settings
\`\`\`

| 명령어 | 용도 |
|--------|------|
| `npm start` | Expo 개발 서버 시작 |
| `npm test` | Jest 테스트 실행 |
| `npm run lint` | ESLint 자동 수정 |
| `npm run type-check` | TypeScript 검증 |
| `npm run build:production` | 프로덕션 빌드 |

## Common Tasks
1. **화면 추가**: src/screens에 생성 후 navigation에 등록, 테스트 작성
2. **커스텀 훅 추가**: src/hooks에 생성, cleanup 함수 포함, 테스트 작성
3. **서비스 추가**: src/services에 싱글턴 패턴으로 생성
```

```markdown
import React, { useEffect, useCallback } from 'react';
import { View, Text, StyleSheet } from 'react-native';

interface StationInfoProps {
  stationName: string;
  lineNumber: number;
  onRefresh: () => void;
}

export const StationInfo: React.FC<StationInfoProps> = ({
  stationName,
  lineNumber,
  onRefresh,
}): React.ReactElement => {
  useEffect(() => {
    const interval = setInterval(onRefresh, 30000);
    return () => clearInterval(interval); // cleanup 필수
  }, [onRefresh]);

  return (
    <View style={styles.container} testID="station-info">
      <Text style={styles.stationName}>{stationName}</Text>
      <Text style={styles.lineNumber}>{lineNumber}호선</Text>
    </View>
  );
};
```

```markdown
## Firestore Pattern
\`\`\`tsx
import { collection, onSnapshot, query, where } from 'firebase/firestore';
import { db } from '@services/firebase';

const useStationData = (stationId: string): StationData | null => {
  const [data, setData] = useState<StationData | null>(null);

  useEffect(() => {
    const q = query(
      collection(db, 'stations'),
      where('id', '==', stationId)
    );
    const unsubscribe = onSnapshot(q, (snapshot) => {
      // 데이터 처리
    });
    return () => unsubscribe(); // cleanup 필수
  }, [stationId]);

  return data;
};
```
