# Save & Compact

Dev Docs에 현재 세션의 컨텍스트를 저장하고 /compact를 실행합니다.

## When to Use

- 컨텍스트가 20% 이하로 떨어졌을 때
- 세션을 종료하기 전
- 대규모 작업 완료 후 정리할 때

## Instructions

다음 단계를 순서대로 실행하세요:

### Step 1: Dev Docs 업데이트

**1.1 활성 프로젝트 확인**
```bash
ls dev/active/
```

**1.2 각 활성 프로젝트의 Context 문서 업데이트**

`*-context.md` 파일에서:
- "Last Updated" 타임스탬프를 현재 시각으로 갱신
- 새로 결정된 사항 추가
- "현재 이슈" 섹션 업데이트
- "다음 단계" 수정
- Session History에 새 세션 요약 추가:
  - 이번 세션에서 완료한 작업
  - 발견된 블로커
  - 다음 우선 조치 사항

**1.3 Tasks 문서 업데이트**

`*-tasks.md` 파일에서:
- 완료된 항목을 `[x]`로 표시
- 발견된 새 작업 추가
- 완료 카운트 업데이트
- 필요시 우선순위 재정렬

### Step 2: 업데이트 확인

사용자에게 업데이트 요약을 보여주세요:

```markdown
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📝 DEV DOCS UPDATED
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ Context 업데이트됨
   - 타임스탬프: [현재 시각]
   - 세션 요약 추가됨

✅ Tasks 업데이트됨
   - 완료: X개
   - 신규: Y개

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### Step 3: /compact 및 /clear 실행

Dev Docs 업데이트가 완료되면:

1. `/compact`을 실행하여 컨텍스트를 압축합니다.
2. `/clear`를 실행하여 대화 기록을 비웁니다 (새 세션 시작).

```
/compact
```

compact 완료 후:

```
/clear
```

## Checklist

- [ ] 타임스탬프 업데이트됨
- [ ] 세션 요약 추가됨
- [ ] 완료 작업 체크됨
- [ ] 새 작업 추가됨
- [ ] 다음 단계 명확히 기록됨
- [ ] 블로커 문서화됨
- [ ] /compact 실행됨
- [ ] /clear 실행됨 (새 세션 시작)

## Session Summary Template

Context 문서에 추가할 세션 요약 템플릿:

```markdown
### [날짜] [시간] ([작업 주제])
- **세션 유형**: [개발/버그 수정/리팩토링/문서화/etc.]
- **완료 작업**:
  - [완료한 작업 1]
  - [완료한 작업 2]
- **원복 작업** (있는 경우):
  - [원복한 작업]
- **블로커**: [블로커 또는 "없음"]
- **다음 우선 조치**:
  - [다음 할 일 1]
  - [다음 할 일 2]
```

## Tips

- 세션 요약은 간결하지만 다음 세션에서 이해할 수 있도록 구체적으로!
- 컴파일 에러나 주요 변경사항은 꼭 기록
- 블로커가 있으면 해결 방법도 함께 기록
