# TDD Command

tdd-guide 에이전트를 호출하여 테스트 주도 개발을 진행합니다.

## 절차

1. **인터페이스 정의** - 타입/인터페이스 먼저
2. **실패 테스트 작성** (RED)
3. **최소 구현** (GREEN)
4. **리팩토링** (REFACTOR)
5. **커버리지 검증** - 80%+ 확인

> 3개 이상 파일 변경 예상이면 `/plan` 먼저 (HARD-GATE)

## TDD 사이클

```
RED → GREEN → REFACTOR → REPEAT

RED:      실패하는 테스트 작성
GREEN:    통과하는 최소 코드 작성
REFACTOR: 코드 개선, 테스트 유지
REPEAT:   다음 기능/시나리오
```

## 사용 시점

- 새 기능 구현
- 버그 수정 (버그 재현 테스트 먼저)
- 리팩토링
- 핵심 비즈니스 로직

## 커버리지 요구사항

- **80% 최소** 모든 코드
- **100% 필수**: 금융 계산, 인증, 보안, 핵심 비즈니스 로직
