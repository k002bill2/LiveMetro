# Plan Command

planner 에이전트를 호출하여 코드 작성 전 구현 계획을 수립합니다.

## 절차

1. **요구사항 재정리** - 무엇을 만들어야 하는지 명확히
2. **리스크 식별** - 잠재적 문제와 차단 요소
3. **단계별 계획** - 구현을 단계로 분해
4. **확인 대기** - 사용자 승인 전 코딩 시작 금지

## 사용 시점

- 새 기능 시작
- 아키텍처 변경
- 복잡한 리팩토링
- 3개 이상 파일 변경 예상 (Golden Principle: HARD-GATE)

## 동작

planner 에이전트가:
1. 요청 분석 후 요구사항을 명확하게 재정리
2. 구현 단계 분해 (3-6 단계)
3. 의존성 식별
4. 리스크 평가 및 복잡도 추정
5. 계획 제시 후 사용자 확인 대기

## 후처리

사용자가 계획 확인 시 `prompt_plan.md`에 저장하여 다음 세션에서 재사용.
